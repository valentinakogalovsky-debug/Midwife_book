# MVP-сайт Михаль Когаловски

Сайт акушерки Михаль Когаловски и книги «Роды с холодной головой» для русскоязычных женщин, которые готовятся к беременности и родам в Израиле.

## Цель MVP

MVP должен помочь посетительнице:

- понять, кто такая Михаль Когаловски и чем она полезна русскоязычным женщинам в Израиле;
- узнать о книге «Роды с холодной головой»;
- посмотреть подробное описание книги;
- оставить заявку на заказ книги;
- выбрать удобный способ связи: WhatsApp или Telegram.

На этапе MVP сайт не принимает оплату и не оформляет покупку автоматически. После отправки заявки Михаль или менеджер связывается с посетительницей вручную.

## Архитектура

Базовая схема:

```text
Next.js / Vercel -> Tally -> Google Sheets -> уведомление менеджеру -> ручная продажа в WhatsApp или Telegram
```

В рамках MVP не используется собственный сервер для заявок, SQLite, PostgreSQL, CRM, личный кабинет, онлайн-оплата или автоматическая выдача электронной книги.

## Стек

- Next.js
- React
- TypeScript
- Tailwind CSS
- Tally для формы заявки
- Google Sheets для хранения заявок
- Google Apps Script, Make или Zapier для уведомлений менеджеру
- Vercel для публикации
- GitHub для хранения исходного кода

## Страницы

- `/` — основной лендинг
- `/book` — подробная страница книги
- `/privacy` — политика конфиденциальности и обработки персональных данных
- `/thank-you` — страница благодарности после отправки заявки

Страница `/thank-you` должна быть закрыта от индексации.

## Основной пользовательский сценарий

1. Пользователь открывает сайт.
2. Читает о Михаль и книге.
3. Переходит на страницу книги или сразу к форме заявки.
4. Заполняет форму Tally.
5. Выбирает предпочтительный способ связи: WhatsApp или Telegram.
6. Подтверждает согласие на обработку персональных данных.
7. Отправляет заявку.
8. Заявка попадает в закрытую Google-таблицу.
9. Менеджер получает уведомление о новой заявке.
10. Пользователь попадает на страницу благодарности или видит сообщение об успешной отправке.
11. Михаль или менеджер связывается с пользователем вручную.

## Форма заявки

Форма создаётся и валидируется в Tally.

Поля формы:

- имя — обязательное поле;
- номер телефона — обязательное поле;
- email — необязательное поле;
- предпочтительный способ связи: WhatsApp или Telegram — обязательное поле;
- Telegram username — условное поле, обязательное только при выборе Telegram;
- согласие на обработку персональных данных — обязательный флажок.

В MVP не добавляются видимые поля «город», «комментарий», «формат книги» или «стоимость». Эти детали уточняются вручную после заявки.

## Валидация формы

Валидация настраивается в Tally:

- имя: обязательное, рекомендуемая длина 2–80 символов;
- телефон: обязательный, допускаются цифры, пробелы, `+`, скобки и дефисы;
- email: проверка формата, если поле заполнено;
- способ связи: обязательный выбор WhatsApp или Telegram;
- Telegram username: отображается и становится обязательным только при выборе Telegram;
- согласие на обработку данных: обязательно до отправки.

Отдельный серверный обработчик и серверная валидация в коде сайта для MVP не требуются.

## Google Sheets

Каждая валидная заявка должна попадать в закрытую Google-таблицу отдельной строкой.

Рекомендуемые столбцы:

- дата и время отправки;
- имя;
- телефон;
- email;
- предпочтительный способ связи;
- Telegram username;
- подтверждение согласия на обработку персональных данных;
- статус обработки;
- комментарий менеджера.

Ссылка на таблицу, приватные ключи, токены и реальные заявки не должны храниться в репозитории.

## Уведомление менеджеру

После появления новой строки в Google Sheets менеджер должен получать уведомление.

Возможные варианты:

- Google Apps Script — простой и дешёвый вариант внутри Google Sheets;
- Make — удобный no-code сценарий;
- Zapier — альтернативный no-code сценарий;
- Telegram Bot API — уведомление в закрытый чат менеджера;
- email-уведомление — минимальный резервный вариант.

Рекомендуемый MVP-вариант: Google Sheets + Google Apps Script или Make.

Пример уведомления:

```text
Новая заявка на книгу
Имя: Анна
Телефон: +972...
Способ связи: WhatsApp
Telegram: -
Время: 2026-07-10 12:30
```

## WhatsApp и Telegram

После отправки заявки пользователь может увидеть на `/thank-you` кнопки:

- «Написать в WhatsApp»
- «Открыть Telegram»

Эти кнопки не заменяют форму. Форма должна сначала сохранить заявку, чтобы не потерять пользователя, если он откроет мессенджер и не напишет.

Рекомендуемый подход:

1. Пользователь отправляет форму.
2. Заявка сохраняется в Google Sheets.
3. Пользователь видит страницу благодарности.
4. На странице благодарности есть кнопка для быстрого перехода в выбранный мессенджер.
5. Менеджер также получает уведомление и может связаться вручную.

## Переменные окружения

Создать файл `.env.local` на основе `.env.example`.

Предварительный список переменных:

```env
NEXT_PUBLIC_TALLY_FORM_ID=
NEXT_PUBLIC_WHATSAPP_LINK=
NEXT_PUBLIC_TELEGRAM_LINK=
```

В репозиторий должен попасть только `.env.example` без реальных значений. Файлы `.env.local` и production-секреты не коммитятся.

## Локальный запуск

После создания Next.js-проекта:

```bash
npm install
npm run dev
```

Локальный адрес:

```text
http://localhost:3000
```

Проверить маршруты:

- `http://localhost:3000/`
- `http://localhost:3000/book`
- `http://localhost:3000/privacy`
- `http://localhost:3000/thank-you`

## Настройка Tally

1. Создать форму Tally для заказа книги.
2. Добавить поля из раздела «Форма заявки».
3. Настроить обязательность полей.
4. Настроить условную логику для Telegram username.
5. Добавить флажок согласия на обработку данных со ссылкой на `/privacy`.
6. Настроить сообщение об успешной отправке или redirect на `/thank-you`.
7. Подключить интеграцию с Google Sheets.
8. Выполнить тестовую отправку.
9. Проверить, что строка появилась в правильной Google-таблице.

## Деплой на Vercel

1. Подключить GitHub-репозиторий к Vercel.
2. Указать production-переменные окружения.
3. Выполнить первый деплой.
4. Проверить страницы `/`, `/book`, `/privacy`, `/thank-you`.
5. Проверить форму на опубликованном домене.
6. Проверить, что заявка попадает в Google Sheets.
7. Проверить, что менеджер получает уведомление.

## Домен

Финальный домен выбирается и регистрируется в зоне `.com` или `.co.il`.

После регистрации:

- подключить домен к проекту Vercel;
- настроить DNS у регистратора;
- включить HTTPS;
- выбрать основной вариант адреса: с `www` или без `www`;
- настроить redirect со второго варианта;
- проверить все страницы и форму на финальном домене.

## SEO и техническая подготовка

Для MVP нужно:

- указать русский язык сайта;
- настроить уникальные `title` и `description` для `/` и `/book`;
- добавить Open Graph-изображение 1200×630;
- добавить favicon;
- добавить `sitemap.xml`;
- добавить `robots.txt`;
- закрыть `/thank-you` от индексации;
- использовать один `h1` на каждой содержательной странице;
- добавить `alt` для смысловых изображений;
- проверить контраст, фокус клавиатуры и мобильные размеры кнопок;
- оптимизировать изображения.

## Материалы, которые нужно подтвердить до публикации

- финальная цена книги и валюта;
- доступные форматы книги;
- условия оплаты и получения;
- контакт WhatsApp;
- Telegram username или ссылка;
- email и социальные сети;
- фотографии Михаль с разрешением на публикацию;
- обложка книги;
- логотип в SVG/PNG и favicon;
- отзывы и согласие на публикацию;
- данные владельца сайта для политики конфиденциальности;
- финальный юридический текст `/privacy`;
- домен.

## Что не входит в MVP

- онлайн-оплата;
- автоматическая доставка электронной книги;
- собственный backend для заявок;
- SQLite или другая собственная база данных;
- CRM;
- личный кабинет;
- авторизация;
- админ-панель;
- блог;
- мультиязычная версия;
- запись на консультацию;
- интеграция с календарём;
- полноценный WhatsApp-бот;
- полноценный Telegram-бот для продаж.

## Критерии готовности

MVP можно считать готовым, если:

- реализованы страницы `/`, `/book`, `/privacy`, `/thank-you`;
- форма Tally встроена на главной странице;
- обязательные поля валидируются;
- Telegram username обязателен только при выборе Telegram;
- без согласия на обработку данных форма не отправляется;
- валидная заявка появляется в Google Sheets;
- менеджер получает уведомление о новой заявке;
- пользователь видит понятный результат отправки;
- сайт адаптирован для desktop и mobile;
- проект опубликован на Vercel;
- подключён домен;
- README актуален;
- в репозитории нет секретов и реальных персональных данных.
